# Введение

Система плагинов — это ключевая часть Rspress. Она позволяет расширять Rspress в процессе сборки сайта. Сначала рассмотрим общую архитектуру Rspress.

Общая архитектура Rspress показана на рисунке ниже:

![Архитектура Rspress](https://assets.rspack.rs/rspress/assets/rspress-architecture.png)

Rspress состоит из двух основных частей: **Node-часть** и **браузерный runtime**. Система плагинов позволяет расширять обе части. В частности, плагины могут:

- Расширять [**компиляцию Markdown/MDX**](/plugin/system/plugin-api#markdown), добавляя плагины [`remark`](https://github.com/remarkjs/remark) или [`rehype`](https://github.com/rehypejs/rehype).
- [**Добавлять пользовательские страницы**](/plugin/system/plugin-api#addpages) поверх стандартной маршрутизации Rspress, например маршрут `/blog` с собственным списком публикаций.
- [**Настраивать поведение сборщика**](/plugin/system/plugin-api#builderconfig), изменяя конфигурацию [Rsbuild](https://rsbuild.rs) или подключая плагины Rspack/Rsbuild.
- [**Расширять метаданные страниц**](/plugin/system/plugin-api#extendpagedata). Rspress вычисляет метаданные вроде `title` и `description` для каждой страницы; плагины могут расширять эту логику, а тема может получать результат через [usePageData](/ui/hooks/use-page-data).
- Выполнять [**пользовательскую логику**](/plugin/system/plugin-api#beforebuildafterbuild) до и после сборки, например закрытие обработчиков событий после завершения сборки.
- [**Добавлять глобальные компоненты**](/plugin/system/plugin-api#globaluicomponents). Поскольку Rspress рендерится на React, плагины могут добавлять глобальные компоненты, например кнопку «наверх» или глобальные компоненты с побочными эффектами.
